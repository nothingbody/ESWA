# Reproducibility Notes

This document records the reproducibility boundary for the public code artifact.

## Environment

- Python: 3.11 or 3.12
- Platform used during development: CPU-only Windows and Linux-compatible
  scripts
- Core external packages: `scipy`, `matplotlib`, `Pillow`, `python-docx`,
  `pytest`
- Optional package: `ortools` for the engineering baseline

Install:

```bash
python -m pip install -r requirements.txt
```

## Included Artifacts

The repository includes:

- source code under `src/`;
- experiment drivers and analysis scripts under `experiments/`;
- unit tests under `tests/`;
- processed result tables under `results/tables/`;
- generated result figures under `results/figures/` and `docs/eswa_figures/`;
- anonymized manuscript materials under `docs/`.

The repository excludes:

- raw benchmark instances;
- runtime logs and status markers;
- local or remote server paths;
- author-identifying title page and cover letter;
- binary DOCX exports, because those can carry local document metadata.

## Data Placement

Place raw public benchmark instances under:

```text
data/dimacs_vrptw/raw/VRPTWController-master/Instances/Solomon/
data/li_lim/hexaly/raw/pdptw/instances/
data/homberger/selected_200_400/
```

Then generate the local BKS tables:

```bash
python experiments/write_bks_tables.py
```

## Minimal Verification

```bash
python -m pytest -q
python experiments/statistical_tests.py
python experiments/fast_slow_runtime_analysis.py
```

Expected behavior:

- the pytest suite passes using embedded sample instances;
- `results/tables/eswa_statistical_tests.csv` contains the paired Wilcoxon
  tests used by the manuscript;
- `results/tables/fast_slow_runtime_summary.csv` and
  `results/tables/fast_slow_difficulty_groups.csv` are regenerated from
  existing processed result tables.

## Full Experiment Rerun

After placing benchmark data, use the batch scripts as references:

```bash
bash experiments/rerun_full_pipeline.sh
```

On Windows, run the Python commands inside that shell script manually or use a
Linux shell environment. The full run is CPU-only but can take substantial time.

## Statistical Tests

The manuscript reports paired Wilcoxon tests from:

```bash
python experiments/statistical_tests.py
```

OR-Tools is not included in the paired tests because its 30-second engineering
reference run did not return feasible verified outputs for all 56 Solomon
instances. A paired test requires the same instances to be solved feasibly by
both compared methods.

## Fast-Slow Runtime Evidence

The fast-slow runtime summary is regenerated by:

```bash
python experiments/fast_slow_runtime_analysis.py
```

The difficulty grouping is based on the initial nearest-neighbor vehicle gap in
the processed Solomon ablation data. The current manuscript reports the
corresponding low, medium, and high groups from the generated CSV table.
